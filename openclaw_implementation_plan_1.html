The Claude Code CLI Trick: Instead of paying Anthropic API directly for complex tasks, OpenClaw can call claude code via bash as a subprocess. This uses your existing Pro subscription credits. The OpenClaw community calls this the "bash delegation" pattern — Gemini handles orchestration, Claude Code handles the heavy coding. 
// ~/.openclaw/openclaw.json — 3-tier model routing config
{
  "models": {
    "providers": {
      "anthropic": {
        "apiKey": "$ANTHROPIC_API_KEY",  // env var, never hardcode
        "models": [{ "id": "claude-sonnet-4-6" }]
      },
      "google": {
        "apiKey": "$GEMINI_API_KEY",
        "models": [
          { "id": "gemini-3-flash" },
          { "id": "gemini-2.0-flash" }
        ]
      },
      "ollama": {
        "baseUrl": "http://localhost:11434/v1",
        "models": [{ "id": "qwen3:32b" }]
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "google/gemini-3-flash",      // daily driver
        "thinking": "anthropic/claude-sonnet-4-6", // complex tasks
        "fallback": "ollama/qwen3:32b"           // free local
      }
    }
  }
}

Security Hardening Checklist
OpenClaw has had 9+ CVEs and 42,665 exposed instances in its first 2 months. Don't be one of them.
⚠️ Critical: OpenClaw's Canvas Host defaults to binding 0.0.0.0, meaning any device on your local network can reach your interface. This is tracked as GitHub Issue #5263 and was closed as "not planned." You must fix it manually.
1. Run in Docker (Non-Negotiable)
  • Containerize OpenClaw with restricted volume mounts
  • Only mount ~/projects and ~/openclaw-workspace — never /
  • Use Docker's network isolation to control outbound traffic
  • Consider NemoClaw (Nvidia's security add-on) for OpenShell sandboxing
2. API Key Management
  • Never hardcode keys in openclaw.json — use environment variables
  • Set spending limits on Anthropic console ($30/mo cap to start)
  • Use a dedicated Google account for Gemini API — not your primary Gmail
  • Rotate API keys monthly
3. Network & Access Control
  • Bind Canvas Host to 127.0.0.1 — not 0.0.0.0
  • Never run OpenClaw as root — create a dedicated macOS user
  • Use file allowlists in SOUL.md to restrict directory access
  • Monitor logs: tail -f ~/.openclaw/logs/gateway.log
4. Skill Vetting
  • Only install skills from the official ClawHub registry
  • Review source code before installing any community skill
  • Cisco found data exfiltration in unvetted third-party skills
  • Start with 5-6 core skills, expand only after verification
Minimum Safe Skills to Start: Calendar (read-only), Gmail (read-only), Exa Search, GitHub (read-only), Telegram Bot. Add write permissions only after 2 weeks of stable operation.
